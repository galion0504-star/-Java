Лабораторная работа №4: Исключения и работа с ресурсами в Java

Описание

Данная работа посвящена углубленному изучению механизмов обработки исключений в Java. Основная цель — научиться изолировать потенциально опасный код, корректно обрабатывать стандартные ошибки и создавать собственные классы исключений для специфических бизнес-задач.

Ключевые концепции

Иерархия Exception: Понимание различий между проверяемыми (Checked) и непроверяемыми (Unchecked) исключениями.

Блоки Try-Catch-Finally: Использование try для мониторинга кода, catch для перехвата ошибок и finally для гарантированного освобождения ресурсов.

Собственные исключения: Создание классов-наследников Exception для более точного описания проблем в логике программы.

Логирование: Механизм сохранения информации об ошибках в текстовый файл для последующего анализа.

Содержание работы:

1. Вычисление среднего арифметического (ArrayAverage)

  Программа рассчитывает среднее значение элементов массива с многоуровневой защитой:

  Обработка ArrayIndexOutOfBoundsException на случай выхода за границы массива (введено меньше элементов, чем указанный размер).

  Обработка NumberFormatException при попытке преобразования нечисловых данных в число.

  Блок finally для гарантированного закрытия ресурса Scanner.

2. Безопасное копирование файлов (FileCopy)
 
  Комплексная задача по передаче данных между файлами. Программа реализует полный цикл безопасного ввода-вывода:
  
  Открытие и закрытие: Гарантированное освобождение файловых дескрипторов в блоке finally (чтение и запись посимвольные).
  
  Чтение и запись: Обработка IOException и FileNotFoundException, возникающих в процессе потоковой передачи данных.
  
  Проверка прав доступа и физического наличия файлов перед началом операции.

3. Пользовательские исключения и логирование (Вариант 1: деление на ноль)

  Реализован механизм кастомной обработки числовых данных:
  
  CustomDivisionException: Собственный класс исключения (наследник Exception), который выбрасывается при попытке деления на ноль.
  
  Система логирования: Весь процесс работы программы сопровождается записью в файл exceptions_log.txt. Каждое пойманное исключение фиксируется в лог-файле с указанием типа ошибки и системного сообщения.
