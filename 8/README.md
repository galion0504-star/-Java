Лабораторная работа №8: Аннотации, Stream API и многопоточность в Java

Описание

Проект представляет собой консольное Java-приложение, реализующее расширяемый конвейер для автоматической обработки числовых данных. В рамках работы продемонстрировано совместное использование механизмов метапрограммирования (аннотаций и рефлексии), функционального стиля обработки коллекций (Stream API) и высокоуровневой многопоточности (ExecutorService).

Основной стек технологий:

- Java Core: (Аннотации, Reflection API);

- Stream API: (функциональная фильтрация, трансформация и сортировка коллекций);

- Java Concurrency Utilities: (ExecutorService, пулы потоков);

- Java NIO: (Files, Paths для работы с файловой системой).

Содержание и архитектура заданий:

Проект спроектирован по модульному принципу и разделен на несколько логических компонентов, взаимодействующих между собой во время выполнения программы.

1. Создание кастомной аннотации @DataProcessor

Суть задачи: Разработка метаданных-маркеров для гибкого и динамического подключения новых модулей обработки без изменения основного кода системы.

Реализация:

- Аннотация ограничена использованием только на уровне методов (ElementType.METHOD — по умолчанию)

- Жизненный цикл аннотации установлен в RetentionPolicy.RUNTIME, что делает её доступной для анализа в оперативной памяти во время работы приложения

2. Реализация модулей обработчиков (Процессоров)

Суть задачи: Создание независимых классов, содержащих изолированную бизнес-логику трансформации числовых данных с использованием Stream API.

Реализованные процессоры:

1.FilterProcessor: filter() -	Оставляет только числа больше 50;

2.TransformProcessor:	transform() -	Умножает каждое число на 2;

3.SortProcessor:	sort() - Сортирует числа по возрастанию.

Все целевые методы внутри классов помечены разработанной аннотацией @DataProcessor.

3. Разработка многопоточного менеджера данных DataManager

Суть задачи: Координация всего жизненного цикла данных: от загрузки из внешнего источника до параллельного распределения задач по потокам.

- Загрузка и сохранение (loadData(), saveData()) -- Использование Files.readAllLines() и BufferedWriter для работы с файлами;

- Регистрация процессоров	(registerDataProcessor()) -- Добавление объектов-обработчиков в список;

- Рефлексия	(processData()) -- Динамическое сканирование методов с аннотацией @DataProcessor и их вызов через method.invoke();

- Многопоточность	(ExecutorService) -- Каждый найденный метод оборачивается в задачу и отправляется в пул потоков.


Ключевые инструменты и методы:

- @interface (Аннотация) -- Инструмент добавления метаданных, позволяющий размещать код для его последующего автоматического анализа сторонними компонентами;

- Reflection API (Method, invoke) -- Механизм, позволяющий программе исследовать собственную структуру классов «на лету» и динамически вызывать методы объектов без жесткого прописывания их имен в коде;

- Stream API (stream(), filter(), map(), sorted()) -- Декларативный стиль манипуляции данными, упрощающий агрегацию, фильтрацию и сортировку коллекций;

- ExecutorService -- Высокоуровневый интерфейс пула потоков, оптимизирующий распределение параллельных задач обработки;

- Future.get() -- Метод, блокирующий выполнение главного потока до получения результата от асинхронной задачи.
