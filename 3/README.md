Лабораторная работа №3: Хэш-таблицы и обобщения в Java

Описание

Данная работа посвящена изучению структуры данных «Хэш-таблица», механизмов разрешения коллизий и практическому применению дженериков (Generics) для создания универсальных контейнеров данных. Проект разделен на две части: реализацию собственной хэш-таблицы с нуля и использование встроенной коллекции HashMap для решения прикладной задачи.

Хэш-таблица: Структура данных, хранящая пары «ключ-значение». Использует хэш-функцию для вычисления индекса в массиве, что обеспечивает быстрый доступ к элементам.

Метод цепочек: Способ разрешения коллизий, при котором каждый индекс массива содержит LinkedList (связный список) элементов с одинаковым хэш-кодом.

Дженерики (<K, V>): Позволяют создавать класс HashTable, который может работать с любыми типами данных (например, String, Integer или пользовательские объекты), обеспечивая типобезопасность на этапе компиляции.
Реализация заданий

1. Универсальная хэш-таблица (HashTable<K, V>)

Реализован собственный класс, имитирующий поведение стандартных коллекций.

Основные методы:

put(K key, V value) — вставка пары. Если ключ уже существует, значение обновляется.

get(K key) — поиск значения по ключу за константное время (в среднем).

remove(K key) — удаление элемента из цепочки.

size() и isEmpty() — методы для контроля состояния коллекции.
